    Perlの動的型付け #

     静的型付け言語に慣れていると、動的型付け言語は不安に感じる。

    暗黙の型変換 #

     Perl では、必要に応じて変換されるので、文字列か数か気にする必要はない。+ 演算子を使うと、Perl は自動的に数として加算される。

    + 演算子 # 

    my $x = "123";
my $y = "456";
print $x + $y;

    my $x = "123";
my $y = 456;
print $x + $y;

    実行結果 # 

    579

    “a” + “b”　は？ #

     + 演算子が両辺を数として扱う。“a"も"b"も数として判断できないので、その場合Perlは 0 として扱う。(use warnings; を書くか、オプション -w をつけると、警告を出してくれる。)

    my $x = "a";
my $y = "b";
print $x + $y;

    実行結果 # 

    0

    警告を出してみる #

    プログラム自身にチェックさせる # 

    use strict; // 厳しいプログラムのチェック
use warnings; // 詳しい警告を出力

    use strict;
use warnings;

my $x = "a";
my $y = "b";
print $x + $y;

    実行結果 # 

    Argument "b" isn't numeric in addition (+) at src/main.pl line 6.
Argument "a" isn't numeric in addition (+) at src/main.pl line 6.
0

     変数 “b” および “a” が数値として扱われるべき箇所で、実際には数値でないという警告が出される。

    . 演算子 #

     ドット(.)による、文字列連結は、他のプログラミング言語に慣れていると、違和感を感じてしまう。

    my $x = "123";
my $y = "456";
print $x . $y;

    my $x = "123";
my $y = 456;
print $x . $y;

    実行結果 # 

    123456
